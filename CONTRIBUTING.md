# Contributing to KXEC

Thank you for your interest in contributing to KXEC! All contributions are welcome — whether you're fixing a typo, improving the FAQ, tightening legal language, or suggesting new features.

## How to Contribute

1. **Fork** this repository and create a new branch for your changes.
2. Make your changes on that branch.
3. Open a **Pull Request** (PR) against `main` with a clear description of what you changed and why.

That's it! There's no complex setup required.

## Types of Contributions

- **Documentation & FAQ fixes:** Corrections to the README, FAQ, or other docs are always appreciated.
- **Legal improvements:** If you spot a loophole or ambiguity in the license text, please open an issue or PR. Use the "Hole in FAQ or license" issue template.
- **Feature requests / ideas:** Use the "Feature request" issue template.

## First-time Contributors

New to open source? No problem! Here are some easy places to start:

- Fix a typo or grammar issue in any `.md` file.
- Improve an FAQ answer.
- Add a missing question to the FAQ.

## Spell Checking

We use [CSpell](https://cspell.org/) to check spelling. You can run it locally before submitting your PR (requires `npm`):

```bash
npx cspell "**"
```

CSpell also runs automatically on all PRs via GitHub Actions. Your PR should pass the spelling check before it is merged.

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md). We want this to be a welcoming space for everyone.
